## Фаза 1 — Оболочка и сайдбар (см. скрины 2, 3, 4)
Убрать верхний бар и нижний футер. Единственная навигация — **левый сайдбар**, сверху вниз:
1. **Мини-месяц** (месяц/год, `< >`, сегодня — акцентом).
2. **AI Chat — выделенная пилюля** (иконка поиска + «AI Chat» + бейдж-шорткат `⌘/`), визуально ярче
   навигации. Справа — **кнопка-стрелка** (открывает компактный плавающий чат-оверлей, не таб).
3. **Search / command** бар (`⌘K`).
4. **Навигация:** `Calendar` (иконка + лейбл + **дата справа**, как Motion «Fri Jul 10») ·
   `Projects & Tasks` (+ бейдж просрочки, Фаза 5) · `Focus` (живой таймер: Deep Work — остаток; Pomodoro — шкала).
   Inbox и AI Agenda — **удалить из кода**.
5. **Calendars** секция (My calendars / пусто-состояние).
6. **Низ сайдбара:** аватар (клик → профиль-дропдаун) · шестерёнка Settings · **sparkle-кнопка AI**
   (открывает чат). **Sparkle-иконку использовать как основную иконку AI** во всём приложении.
- **Профиль-дропдаун** (клик по аве, см. скрин 3): email вверху · Settings · Help · переключатель темы
  **Light / Dark / System** (три тайла) · Log out.
- Убрать «блоки»-обводки где не нужно; фон везде `#1B1D1E`.
Коммит: `feat(fd-1): flowday shell — sidebar, ai entries, profile menu`.

## Фаза 2 — Календарь как на скрине 1 + баг-фиксы
- Привести вид к Motion (скрин 1): плоский `#1B1D1E`, чёткие линии, компактные лейблы часов,
  «сегодня» — акцентная плашка на числе, текущий час — тонкая акцентная линия.
- **Event-чипы по календарю** (цвет из календаря): тайтл + время под ним; задачи снизу дня — компактные
  строки с точкой приоритета.
- **БАГ:** при добавлении задачи ломаются цвета — почини (цвет из календаря/priority, не сбрасывать).
- **БАГ:** ховер по нерабочим часам ломает цвета — почини hover нерабочих ячеек.
- Клик по пустому слоту → **простая event-модалка** с предзаполненным временем (без поповера).
Коммит: `fix(fd-2): calendar look (screen1) + color/hover bugs`.

## Фаза 3 — Единый стиль + тултипы

## Фаза 4 — Settings (скрин 6 плохой) + Кастомизация + AI-провайдеры
1. Переделать Settings **как Motion**: двухпанельно, выровнять контролы, довести палитру, фон `#1B1D1E`
   (панель НЕ `#262627`), карты-строки `#262627`, helper-текст, без лишних обводок.
2. **Раздел «Customization»** (per-user в БД):
   - **Акцентный цвет** — палитра свотчей + выбор, **меняется вживую** (пишет `--accent`).
   - Оттенок фона, **плотность**, ширина сайдбара, скругления, шрифт, стиль event-чипов, анимации on/off.
   - **3 визуальных темы** (разный вид, не только цвет) — **заготовки, сейчас locked**: кликаются, но
     заблокированы, с бейджем **«Coming soon»**. Оставь чистую точку расширения под будущие темы.
3. **Раздел «AI»:** провайдеры **Anthropic · OpenAI · Grok (xAI) · GLM (z.ai)**. Ввод **API-ключа**,
   хранение **per-user зашифровано** (`AI_ENCRYPTION_KEY`); OAuth — заглушка «soon» где нет. Grok/GLM —
   OpenAI-совместимые (кастомный `baseURL`). Выбор провайдера/модели по умолчанию. Soul-пресет:
   **(1) краткий деловой** / **(2) дружелюбный ADHD-коуч**. Ключи только на сервере.
Коммит: `feat(fd-4): settings + customization (live accent, locked themes) + ai providers`.

## Фаза 5 — Projects & Tasks + бейджи просрочки
- Список задач **как Motion** (плотная таблица, лёгкое создание/редактирование, фильтры, пусто-состояние).
  Виды: **List · Board (kanban) · Timeline (Gantt по проектам)**.
- **Projects:** создание/удаление, назначение задач (drag), прогресс, **цвет+иконка** (поля миграцией).
- **Бейджи просрочки** (на нав-пункте `Projects & Tasks` И на карточках проектов):
  - **Оранжевый** — есть задачи с дедлайном сегодня ИЛИ просроченные.
  - **Красный** — есть задачи, просроченные **больше 3 дней**.
  - Формат как скрин 4: значок «‼ N». Считать по задачам проекта/всего.
Коммит: `feat(fd-5): projects & tasks, gantt, overdue badges`.

## Фаза 6 — AI-агент («soul») с управлением приложением
- **Страница AI Chat как Motion**: колонка истории (`Search chats`, `+ New chat`, беседы по датам),
  центр — приветствие + карточки-подсказки, крупный инпут снизу. + **плавающий оверлей-чат** (Фаза 1).
- Стриминг по токенам. **История в БД** (`AiConversation`/`AiMessage`).
- **Без подключённого провайдера → блок ввода + CTA** «Добавь ключ / авторизуйся» → Settings ▸ AI.
- **Тулзы (function-calling)** на существующие действия: `create_task`, `edit_task`, `delete_task`,
  `schedule_task`/`auto_schedule` (дёргает детерминированный движок), `parse_brain_dump`,
  `query_schedule`, `manage_projects`. **Автономный**, но **подтверждение на опасное** (удаление,
  полный перепланинг). Область: задачи/календарь + общий чат. Системный промт — по Soul-пресету.
Коммит: `feat(fd-6): ai agent with app-control tools`.

## Фаза 7 — «Живой» AI на календаре
Когда агент действует на календаре — **анимированный AI-курсор** с sparkle-иконкой «перетаскивает»
задачи в слоты (framer-motion), чтобы выглядело живым. Тактично, под `prefers-reduced-motion` (без курсора).
Коммит: `feat(fd-7): animated ai cursor`.

`prefers-reduced-motion`. 150–220ms ease-out.
